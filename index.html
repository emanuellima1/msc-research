A Design of Experiments Approach to Autotuning the Rust Compiler

Rustc is the official compiler of the Rust Programming Language. It uses the LLVM compiler framework as a back end to generate binaries. LLVM transforms its Intermediate Representation (LLVM IR) by executing a series of optimization passes, that can be toggled by flags, in order to generate better optimized binaries. These passes interfere on metrics such as the execution time. Manually selecting flags to improve the execution time of a program is difficult, error prone and requires expert knowledge. Autotuning methods can automate parts of this process and help to understand the underlying search spaces. In this research we will seek to understand which LLVM flags most impact the execution time of Rust programs, using the Design of Experiments methodology.
